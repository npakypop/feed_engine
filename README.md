# feed_engine

Цей проєкт `feed_engine` призначений для перетворення XML-фіду товарів з майстер-джерела у формат, зручний для Maudau.

## Що робить цей проєкт

- Завантажує XML-фід з основного джерела за URL, заданим у змінній оточення `MASTER_FEED_URL`.
- Парсить категорії, пропозиції (offers), параметри товарів, зображення та опис.
- Очищує текстове наповнення від HTML-тегів та небажаних символів.
- Перекладає назви параметрів і значення з російської на українську, використовуючи кеш для економії запитів.
- Коригує ціни множенням на коефіцієнт `PRICE_MULTIPLIER` (за умовчанням `1.11`).
- Генерує вихідний файл `maudau_feed.xml` у потрібному форматі.

## Як це працює

1. Встановіть залежності з `requirements.txt`:

```bash
pip install -r requirements.txt
```

2. Налаштуйте змінну оточення `MASTER_FEED_URL`, щоб вказати URL на майстер-фід.

3. Запустіть скрипт:

```bash
python main.py
```

4. Результатом буде файл `maudau_feed.xml` у поточній директорії.

## Основні файли

- `main.py` — основний скрипт, який завантажує, парсить, перекладає та генерує фінальний XML.
- `requirements.txt` — список Python-пакетів, потрібних для роботи.
- `feed-rozetka.xml`, `maudau_feed.xml`, `unique_params.txt` — допоміжні файли для тестування та збереження результатів.

## Особливості

- Перетворює параметри товару в українську мову за допомогою словника `PARAM_MAP`.
- Підтримує кешування перекладів, щоб зменшити звернення до перекладацького API.
- Використовує `vendorCode` як ідентифікатор товару, якщо він доступний.
- Мапить категорії відповідно до словника `CATEGORY_MAP`, щоб задати `portal_id` для Maudau.

## Примітки

- Якщо поле `vendorCode` порожнє, як `id` для фіду використовується внутрішній `id` з XML.
- Усі посилання на зображення додаються до файлу, якщо вони починаються з `http`.
- Якщо потрібно змінити логіку, наприклад коефіцієнт ціни або набір параметрів, редагуйте `main.py`.
